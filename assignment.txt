Full Stack Application Development 
 
Classification: Public-SLIIT 
Overview: 
This assignment involves developing a full-stack software application. Students will work in groups of four 
to design and implement the system. The project will be completed in two milestones, allowing progress to 
be reviewed and improved in stages. 
Part 1. A Secure RESTful API Backend 
You will build a backend system based on the specific topic assigned to your group. Each group will receive 
a unique domain (e.g., event management, healthcare system, inventory system, student management, travel 
app backend, etc.). 
Backend Requirements: 
a. Develop a full RESTful API using Express.js (Node.js) 
b. Clearly define the functional requirements of each component (a project should have at least 4 
components) 
c. The backend must include individual components, each of which must fulfill the following 
requirements. 
i.  All data manipulation and business logic must be exposed through properly structured 
REST endpoints following standard HTTP methods and status codes (CRUD operations) 
ii. Additional Feature (use of third-party API) 
iii. MongoDB database integration 
iv. Protected routes & role-based access 
v. Validation and Error Handling 
vi. Clean architecture and best practice 
vii. API documentation (Swagger/Postman) 
Part 2.  A React Frontend Application 
In addition to the backend, each group must develop a React frontend application for their backend. The 
React app must consume the backend API developed in Part A and serve as the user interface for the full -
stack system. Students must also demonstrate how state management, session handling, and deployment 
were handled in the project. \n 
Classification: Public-SLIIT 
Frontend Requirements: 
a. React functional component architecture (React Hooks, Context API or Redux for state 
management) 
b. API Integration 
The frontend must consume the backends’ endpoints, such as (user management, Entity 
management – CRUD, Filters, Pagination, search etc.) 
c. UI/ UX Requirements (Tailwind CSS, Bootstrap etc.) 
d. Session Management 
e. Deployment 
Deployment Requirements: 
Backend Deployment: The RESTful API developed using Express.js (Node.js) must be deployed on a 
cloud hosting platform such as Render, Railway, OR any other platform. 
Frontend Deployment: The React frontend application must be deployed using platforms such as  Vercel, 
Netlify, Firebase Hosting, OR any other platform. 
Deployment Documentation: Students must include a Deployment section in the README.md file 
containing: 
• Backend deployment platform and setup steps 
• Frontend deployment platform and setup steps 
• Environment variables used (without exposing secrets) 
• Live URLs for: 
• Deployed backend API 
• Deployed frontend application 
• Screenshots or evidence of successful deployment. 
Additional Testing Requirements: 
1. Unit Testing: 
• Implement unit tests for individual components and functions to validate their behavior in 
isolation. 
 \n 
Classification: Public-SLIIT 
2. Integration Testing: 
• Conduct integration tests to ensure different parts of the application work together 
seamlessly. This includes testing interactions between controllers, services, and the 
MongoDB database. 
• Test API endpoints to verify correct handling of requests and responses, including error 
scenarios. 
3. Performance Testing: 
• Evaluate the API's performance under various loads to ensure it can handle multiple 
requests simultaneously without significant latency. 
• Tools like Artillery.io (for Express JS applications) can be used for performance testing. 
Evaluation Criteria:  
• Correctness and functionality of the application. 
• Creativity and design implementation. 
• Code quality, including readability and use of best practices. 
• Completeness of documentation. 
• Regularity and informativeness of git commit. 
• Thoroughness of testing. 
• Viva performance.
